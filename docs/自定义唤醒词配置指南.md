# 自定义唤醒词配置指南

本文档介绍如何为 HomeBot 语音交互系统配置自定义唤醒词。

## 概述

HomeBot 使用 sherpa-onnx 的 Keyword Spotting (KWS) 功能实现语音唤醒。唤醒词需要转换为特定的音素/拼音格式才能被识别引擎使用。

## 前置条件

- Python 3.8+
- 已安装 HomeBot 依赖：`pip install -r requirements.txt`

---

## 步骤一：安装 sherpa-onnx

```bash
# 激活虚拟环境（Windows）
venv\Scripts\activate

# 安装 sherpa-onnx
pip install sherpa-onnx
```

### 验证安装

```bash
# 使用 Python 模块方式运行（推荐，避免 Windows 路径编码问题）
python -m sherpa_onnx.cli --help
```

---

## 步骤二：设置原始唤醒词

### 2.1 编辑原始关键词文件

打开 `software/models/wakeup/keywords_raw.txt`，每行添加一个唤醒词：

```text
# 格式: 关键词 [:boost分数] [#阈值] [@别名]
# 中文示例
你好小白 :1.5 #0.3 @你好小白
小艺小艺 :1.2 #0.25 @小艺小艺

# 英文示例
HOME BOT :1.0 #0.3 @HOME_BOT
HELLO ROBOT :1.0 #0.35 @HELLO_ROBOT
```

### 2.2 格式说明

| 字段 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `关键词` | 唤醒词文本（必填） | - | `你好小白` |
| `:boost` | boosting score，提高识别敏感度 | `1.0` | `:1.5` |
| `#threshold` | 触发阈值（0-1），越小越容易触发 | `0.0` | `#0.3` |
| `@别名` | 原始文本别名（中文必填） | - | `@你好小白` |

**注意事项：**
- 英文唤醒词含空格时，用下划线 `_` 替换或在转换后自动处理
- 中文唤醒词**必须**提供 `@别名`
- `#threshold` 建议范围 `0.2-0.4`，太小容易误触发，太大难以触发

---

## 步骤三：转换关键词格式

### 3.1 使用 sherpa-onnx-cli 转换

根据唤醒词语言类型选择合适的 `tokens-type`：

#### 方式 A：纯中文唤醒词（ppinyin）

```bash
cd software

python -m sherpa_onnx.cli text2token \
    --tokens ./models/wakeup/tokens.txt \
    --tokens-type ppinyin \
    ./models/wakeup/keywords_raw.txt \
    ./models/wakeup/keywords.txt
```

#### 方式 B：纯英文唤醒词（phone）

需要准备英文音素表 `en.phone`（lexicon 文件）。

```bash
cd software

python -m sherpa_onnx.cli text2token \
    --tokens ./models/wakeup/tokens.txt \
    --tokens-type phone+ppinyin \
    --lexicon ./models/wakeup/en.phone \
    ./models/wakeup/keywords_raw.txt \
    ./models/wakeup/keywords.txt
```

#### 方式 C：中英文混合（推荐 phone+ppinyin）

支持同时包含中文和英文唤醒词：

```bash
cd software

python -m sherpa_onnx.cli text2token \
    --tokens ./models/wakeup/tokens.txt \
    --tokens-type phone+ppinyin \
    --lexicon ./models/wakeup/en.phone \
    ./models/wakeup/keywords_raw.txt \
    ./models/wakeup/keywords.txt
```

### 3.2 转换结果示例

**输入** (`keywords_raw.txt`):
```text
你好小白 :1.5 #0.3 @你好小白
HOMEBOT :1.0 #0.3 @HOMEBOT
```

**输出** (`keywords.txt`):
```text
n ǐ h ǎo x iǎo b ái :1.5 #0.3 @你好小白
▁HH AH M M BB OW TT :1.0 #0.3 @HOMEBOT
```

## 步骤四：验证配置

### 4.1 检查生成的关键词文件

```bash
# 查看转换结果
type software\models\wakeup\keywords.txt
```

### 4.2 测试唤醒功能

```bash
# 启动语音服务（会自动加载新的关键词）
cd software/src
python -m services.speech_service wakeup
```

### 4.3 调整参数优化识别效果

如果唤醒不灵敏或误触发太多，调整 `keywords_raw.txt` 中的参数后重新转换：

| 问题 | 解决方案 |
|------|----------|
| 唤醒困难 | 降低 `#threshold`（如 `#0.2`）或提高 `:boost`（如 `:2.0`） |
| 误触发多 | 提高 `#threshold`（如 `#0.4`）或降低 `:boost`（如 `:0.8`） |

## 附录

### A. tokens-type 说明

| 类型 | 适用场景 | 说明 |
|------|----------|------|
| `ppinyin` | 纯中文 | 部分拼音（声母+韵母分离） |
| `fpinyin` | 纯中文 | 完整拼音（带声调） |
| `phone` | 纯英文 | 英文音素（需 lexicon） |
| `phone+ppinyin` | 中英文混合 | 英文音素 + 中文拼音 |
| `cjkchar` | 纯中文 | 按字符分割 |
| `bpe` | 多语言 | BPE 子词单元（需 bpe.model） |

### B. 常用命令速查

```bash
# 快速转换（中文）
python -m sherpa_onnx.cli text2token --tokens ./models/wakeup/tokens.txt --tokens-type ppinyin ./models/wakeup/keywords_raw.txt ./models/wakeup/keywords.txt

# 快速转换（中英文混合）
python -m sherpa_onnx.cli text2token --tokens ./models/wakeup/tokens.txt --tokens-type phone+ppinyin --lexicon ./models/wakeup/en.phone ./models/wakeup/keywords_raw.txt ./models/wakeup/keywords.txt
```

## 故障排除

### Q1: 转换后文件为空

**原因**: 输入格式不正确或 tokens 文件不匹配。

**解决**: 检查 `keywords_raw.txt` 格式，确保中文关键词包含 `@别名`。

### Q2: 英文唤醒词无法识别

**原因**: 缺少 `en.phone` lexicon 文件或使用了错误的 tokens-type。

**解决**: 使用 `phone+ppinyin` 类型并提供正确的 lexicon 路径。

### Q3: 中文显示乱码

**原因**: Windows 控制台编码问题。

**解决**: 在 PowerShell 中运行 `chcp 65001` 设置 UTF-8 编码。
